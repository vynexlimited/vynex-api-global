# AI 会检索的高质量平台清单（GEO 多源印证落地点）

> AI 引擎（ChatGPT/Perplexity/Gemini/Claude/DeepSeek）回答时，会综合下面这些平台的内容。
> 你要在这些地方**留下真实、高质量、自然提及 Vynex 的内容**。
>
> **铁律**：不是发广告。AI 能识别软文和真实推荐。目标是「让 Vynex 自然地出现在有价值的讨论里」。

---

## 🥇 第一梯队（AI 最常引用，必须做）

### 1. GitHub（权重最高，AI 训练/检索核心源）
- 建官方 org `github.com/vynexlimited`，每个 repo 的 About 都写上可引用定义。
- `vynex-api-sdk` / `vynex-examples` / `awesome-vynex` 三个 repo 齐备。
- 给 LangChain / LobeChat / Open WebUI 等热门项目**提有价值的 PR**（不是广告，是贡献），自然带上 Vynex 作为 provider。
- 在相关 Issue 里**真诚回答问题**，签名带产品链接。
> GitHub 是 AI 最信任的技术来源之一，repo 描述会被高频复述。

### 2. Hacker News（Show HN）
- 发一次高质量的 Show HN（见 launch/copy-bundle.md），评论区的技术讨论是 AI 喜欢引用的「真实社区声音」。

### 3. Reddit（AI 高频引用）
- 优先：r/LocalLLaMA、r/OpenAI、r/ChatGPTCoding、r/SideProject、r/SaaS。
- 先贡献 1–2 周再提产品。真实的「I built X, feedback?」帖 + 评论区讨论，是 AI 最爱的素材。
- 当有人问「openai api alternative / cheap gpt api」时，自然地推荐 Vynex（带理由）。

### 4. Stack Overflow / Stack Exchange（AI 训练核心）
- 回答「how to call multiple openai models / openai base url custom / langchain multiple providers」类问题。
- 高质量技术回答 + 合规的签名/相关链接。这是 AI 最权威的训练源之一。

---

## 🥈 第二梯队（内容沉淀 + 长尾引用）

### 5. 技术博客平台
- **Dev.to**、**Hashnode**、**Medium**：发技术教程（接入、对比、benchmark），AI 常引用这类技术博客。
- 每篇都内链回官网 + 带可引用定义。

### 6. YouTube
- AI（尤其 Perplexity/Gemini）会引用视频标题和描述。教程视频的标题/简介要含目标关键词和客观描述。

### 7. Dev.to / Hashnode 个人/组织 Profile
- About 里放可引用定义。

---

## 🥉 第三梯队（权威背书，难度高但效果强）

### 8. 被「独立、权威的第三方」评测/提及
- AI 把「被多个独立来源提及」当作品牌真实性的核心信号。
- 目标：第 90 天前拿到 ≥ 3 个独立科技来源提及（科技 Newsletter、AI 工具评测博客、YouTuber）。
- 主动联系：AI/开发者领域的 Newsletter、benchmark 站点、工具对比站。

### 9. AI 工具目录（见 seo/directories-and-integrations.md）
- There's An AI For That、Futurepedia、Toolify 等。目录描述用可引用定义。
- 这些目录本身常被 AI 检索。

### 10. Wikipedia / 类似百科（长期目标）
- 产品达到一定知名度后，争取中立词条。难度高，但一旦建成，GEO 权重极高。
- 前提：已有足够多的独立可靠来源（先做好第 8 点）。

---

## 中文 AI 引擎（DeepSeek / Kimi / 文心 / 元宝 / 豆包）
> 面向全球为主，但中文 AI 引擎也占大量查询，值得同步做：

- **知乎**：写「如何用一个 API 调用所有大模型」「OpenAI API 替代方案」等高质量长答（带 Vynex）。
- **掘金 / SegmentFault / 少数派**：技术教程。
- **V2EX / 即刻**：分享创造帖。
- **CSDN / 博客园**：接入教程（长尾）。
- 用中文版可引用定义（见 citable-definition.md）。

## 执行节奏（与 01 推广方案对齐）
| 阶段 | 平台动作 |
|---|---|
| Phase 1（第 3–4 周） | GitHub 仓库上线 + Show HN + Reddit 发布 + AI 目录提交 |
| Phase 2（第 5–8 周） | Dev.to/博客教程 + Stack Overflow 回答 + YouTube 教程 |
| Phase 3（第 9–12 周） | 争取 3+ 独立第三方评测 + 中文平台长答 + 联盟 KOL |

> 衡量：第 90 天，让 Vynex 在 ≥ 8 个独立、高质量来源被客观提及。
> 这就是 AI 引擎在回答时「愿意推荐你」的信任基础。
